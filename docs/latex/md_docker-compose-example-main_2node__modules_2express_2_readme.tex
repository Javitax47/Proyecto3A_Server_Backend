Before installing, \href{https://nodejs.org/en/download/}{\texttt{ download and install Node.\+js}}. Node.\+js 0.\+10 or higher is required.

If this is a brand new project, make sure to create a {\ttfamily package.\+json} first with the \href{https://docs.npmjs.com/creating-a-package-json-file}{\texttt{ {\ttfamily npm init} command}}.

Installation is done using the \href{https://docs.npmjs.com/getting-started/installing-npm-packages-locally}{\texttt{ {\ttfamily npm install} command}}\+:


\begin{DoxyCode}{0}

View the website at\+: \href{http://localhost:3000}{\texttt{ http\+://localhost\+:3000}}\hypertarget{md_docker-compose-example-main_2node__modules_2express_2_readme_autotoc_md820}{}\doxysubsection{\texorpdfstring{Philosophy}{Philosophy}}\label{md_docker-compose-example-main_2node__modules_2express_2_readme_autotoc_md820}
The Express philosophy is to provide small, robust tooling for HTTP servers, making it a great solution for single page applications, websites, hybrids, or public HTTP APIs.

Express does not force you to use any specific ORM or template engine. With support for over 14 template engines via \href{https://github.com/tj/consolidate.js}{\texttt{ Consolidate.\+js}}, you can quickly craft your perfect framework.\hypertarget{md_docker-compose-example-main_2node__modules_2express_2_readme_autotoc_md821}{}\doxysubsection{\texorpdfstring{Examples}{Examples}}\label{md_docker-compose-example-main_2node__modules_2express_2_readme_autotoc_md821}
